## Planning invariants derived from the code-grounded baseline

These are planning directives derived from the code above rather than new current-state claims:

- No shadow implementation: no second agent loop, no second session authority, no second provider parser stack, no second tool router.
- Embedded is a surface plus backends: build through `FactoryAgentBuilder`, run through `RuntimeSessionAdapter`, persist through `PersistentSessionService` and the store traits.
- The web runtime is a reference for handle tables, event polling, and callback-tool shape, not a justification for browser-style substrate authority on embedded.
- The first architecture-prep seam is provider transport because it is the only clearly missing seam blocking shared provider logic.
