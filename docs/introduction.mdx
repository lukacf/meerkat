Meerkat is a **library-first, modular agent harness** — composable Rust crates that handle state machines, retries, budgets, streaming, tool execution, MCP, and multi-agent coordination.

**Stable** — deterministic state machine, typed errors, compile-time guarantees.
**Fast** — under 10ms cold start, ~20MB memory, single 5MB binary.

## Quick start

Every surface runs the same agent engine. Here is "What is the capital of France?" across all seven:

<Tabs>
  <Tab title="Rust">
    ```toml Cargo.toml
    [dependencies]
    meerkat = "0.1"
    tokio = { version = "1", features = ["full"] }
    ```

    ```rust main.rs
    #[tokio::main]
    async fn main() -> Result<(), Box<dyn std::error::Error>> {
        let result = meerkat::with_anthropic(std::env::var("ANTHROPIC_API_KEY")?)
            .model("claude-sonnet-4-5")
            .run("What is the capital of France?")
            .await?;
        println!("{}", result.text);
        Ok(())
    }
    ```

    See the [Rust SDK reference](/SDK) for the full API.
  </Tab>
  <Tab title="Python">
    ```bash
    pip install meerkat-sdk
    ```

    ```python
    from meerkat import MeerkatClient

    async with MeerkatClient() as client:
        await client.connect()
        result = await client.create_session(prompt="What is the capital of France?")
        print(result.text)
    ```

    See the [Python SDK guide](/sdks/python) for the full API.
  </Tab>
  <Tab title="TypeScript">
    ```bash
    npm install @meerkat/sdk
    ```

    ```typescript
    import { MeerkatClient } from "@meerkat/sdk";

    const client = new MeerkatClient();
    await client.connect();
    const result = await client.createSession({ prompt: "What is the capital of France?" });
    console.log(result.text);
    await client.close();
    ```

    See the [TypeScript SDK guide](/sdks/typescript) for the full API.
  </Tab>
  <Tab title="CLI">
    ```bash
    cargo install --path meerkat-cli
    export ANTHROPIC_API_KEY=sk-...

    rkat run "What is the capital of France?"
    ```

    See the [CLI reference](/CLI) for all flags and options.
  </Tab>
  <Tab title="MCP">
    Expose Meerkat as tools to other AI agents via MCP:

    ```json Claude Code / MCP client config
    {
      "mcpServers": {
        "meerkat": {
          "command": "rkat",
          "args": ["mcp-server"]
        }
      }
    }
    ```

    Then call `meerkat_run` with your prompt:

    ```json MCP tool call
    {
      "name": "meerkat_run",
      "arguments": {
        "prompt": "What is the capital of France?"
      }
    }
    ```

    See the [MCP guide](/surfaces/mcp) for all parameters.
  </Tab>
  <Tab title="REST">
    Start the server, then POST:

    ```bash Start server
    cargo run --package meerkat-rest
    ```

    ```bash curl
    curl -X POST http://localhost:8080/sessions \
      -H "Content-Type: application/json" \
      -d '{"prompt": "What is the capital of France?"}'
    ```

    See the [REST API guide](/surfaces/rest) for all endpoints.
  </Tab>
  <Tab title="JSON-RPC">
    The JSON-RPC surface speaks newline-delimited JSON over stdin/stdout:

    ```bash Start RPC server
    rkat rpc
    ```

    ```json Create session
    {"jsonrpc":"2.0","id":1,"method":"session/create","params":{"prompt":"What is the capital of France?"}}
    ```

    See the [JSON-RPC guide](/surfaces/rpc) for all methods.
  </Tab>
</Tabs>

## Modularity

Pick only what you need. Disabled capabilities return typed errors — no panics, no silent degradation.

<CodeGroup>
```toml Minimal
[dependencies]
meerkat = { version = "0.1", default-features = false, features = ["anthropic"] }
```

```toml With persistence
[dependencies]
meerkat = { version = "0.1", features = ["anthropic", "session-store", "session-compaction"] }
```

```toml Everything
[dependencies]
meerkat = { version = "0.1", features = ["all-providers", "session-store", "session-compaction", "memory-store-session"] }
```
</CodeGroup>
